# Architecture (current)

PandemoniumProtocol is an **attention-management layer for LLM coding agents**: it makes
a codebase searchable as cheap, tagged **references** so an agent can find code *and
related code* ‚Äî then fetch exact source only for what it chose, instead of reading whole
files. Fully local; multi-language; works via CLI or an MCP server, driven by a thin
Claude skill.

> Search returns **cards** (ref + summary + tags, *no code*). The agent picks, then
> `repo_get(ref)` returns exact code. The graph answers "what's related / what's
> affected." Summaries guide retrieval; the code is the source of truth.

## The card / ref model

Every indexed unit is a **card**: `ref`, `scope`, `qualified_name`, `parent`,
`signature`, `summary`, `tags` (6 fixed fields) ‚Äî and the **raw code stored separately**,
fetched on demand. A **ref** is a durable, edit-stable anchor:

```
path                     -> whole file
path::Qualified.Name     -> a symbol (re-resolved live by name; survives edits)
path:start-end           -> line span (fallback)
```

## Index-time pipeline

```
scan (ignore secrets/binaries/.pandemonium/) ‚Üí language + content hash (incremental skip)
  ‚Üí tree-sitter parse ‚Üí symbols           (Python, C++, C#, JS, TS ‚Äî spec-driven)
  ‚Üí SYMBOL cards   (non-parsed files get block coverage)
  ‚Üí heuristic summary + tags
       ‚îî‚îÄ ENRICHMENT override (opt-in): Claude-generated summary + tags (no API key)
  ‚Üí build DESCRIPTOR (NL: summary + signature + qname + tags)   ‚Üê what we embed
  ‚Üí embed descriptor (bge-small, 384-d, normalized)             ‚Üê NOT raw code
  ‚Üí extract GRAPH EDGES (calls/imports/inherits, unresolved, per file)
  ‚Üí store:  SQLite (cards + symbols + relationships + raw code)
            FTS5 (keyword)   LanceDB (vectors)
```

## Query-time / retrieval

```
repo_search(query)
  ‚îú‚îÄ symbol channel   exact/prefix on symbol names
  ‚îú‚îÄ keyword channel  FTS5 bm25
  ‚îî‚îÄ vector channel   bge query-prefixed embedding ‚Üí LanceDB KNN (scope-prefilterable)
        ‚Üí per-channel normalize ‚Üí weighted merge ‚Üí file-exempt dedup ‚Üí CARDS
repo_get(ref, expand=exact|neighbors|file|parent, view=full|signature|head:N|lines:a-b)
                                  ‚Üí exact code (symbol re-found live); surfaces a C++
                                    out-of-line def's "declared in <header>:line" (decl_ref)
repo_context_pack(task, budget)  ‚Üí one budgeted markdown pack (the give-me-everything mode)
```

## Relationship graph ‚Äî "related code"

Edges (`calls`, `imports`, `inherits`) are extracted per file and stored **unresolved**;
names resolve to symbols at **query time** (receiver-aware: `self.m()` ‚Üí the class;
`mod.m()` ‚Üí the module), so call-name collisions don't pollute results and reindexing one
file never invalidates another's edges.

```
repo_graph(ref)      ‚Üí callees, callers, imports, inheritance, members, tests,
                       similar implementations (vector), affects (LLM hypotheses)
repo_impact(ref)     ‚Üí directly + transitively affected callers, affected files, tests
repo_logic_map(topic)‚Üí conceptual flow: relevant symbols + domains + call flow
```
Conservative: only confidently-resolved static edges (precision over recall ‚Äî a wrong "X
affects Y" misleads the agent). Two **logic** layers add semantic relationships:
`similar_to` is grounded (vector neighbours of descriptors); `affects` is **LLM-inferred
hypotheses** (config/data/contract/ordering coupling) stored with confidence + evidence
and clearly labelled ‚Äî generated by in-session subagents (no API key), regenerable.

> **`affects` is a DORMANT capability.** The ingest path (`graph.ingest_affects`, which
> merges subagent-produced shard JSON into `affects` edges) is an API only ‚Äî it is **not
> wired to a CLI command or MCP tool**, and no skill teaches an agent to produce shards. So
> in normal operation **no `affects` edges exist** and the read paths above (in `repo_graph`/
> `repo_edit_plan`/`repo_brief`) render empty. Treat `affects` as experimental/out-of-band
> until a first-class produce‚Üíingest workflow is built; the read code is guarded and inert
> until then.

## Session ledger + staleness

`repo_session` is per-session memory (searched queries, surfaced/fetched refs, edited
files, confirmed facts, ‚Ä¶) ‚Äî auto-filled by `repo_search`/`repo_get` ‚Äî so long sessions
don't rediscover. `repo_changed` reports which fetched files changed since indexing.

## Surfaces

- **CLI:** `init, index, sync, changed, reindex-changed, search, symbol, get, context,
  graph, impact, tests, map, serve-mcp`.
- **MCP (15 tools):** `repo_map, repo_search, repo_symbol, repo_get, repo_brief,
  repo_context_pack (repo_prompt_context), repo_find_tests, repo_graph, repo_impact,
  repo_edit_plan, repo_logic_map, repo_session, repo_changed, repo_reindex_changed` ‚Äî
  read-only except reindex; every call audit-logged.
- **Claude skill:** `.claude/skills/pandemonium/` ‚Äî the operating discipline (search
  cards ‚Üí fetch exact ‚Üí use the graph + ledger ‚Üí coordinate parallel agents ‚Üí verify).

## Storage

- **SQLite** (single WAL connection): cards/symbols/relationships/files (+ the full Part 2
  schema). **FTS5**: keyword. **LanceDB**: vectors (dim auto-detected from the model).
  Single-writer discipline; everything under `.pandemonium/`.

## Key decisions (all measured against a baseline; see `evals/RESULTS.md`)

- **Embed descriptors, not code** ‚Üí +13pt precision@1 (representation only).
- **Claude enrichment** (opt-in) ‚Üí +13pt P@5, +27pt symbol-P@5 (same-corpus A/B).
- **Scope bake-off** ‚Üí symbol-only wins; file/code-on-parsed *hurt or cost* ‚Äî not added.
- **Model bake-off** ‚Üí bge-small beats bge-base and a code model (because we embed NL
  descriptors). Smallest + fastest wins.
- **Graph** ‚Üí query-time receiver-aware resolution; gated on a no-collision test.
